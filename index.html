03Findings
[F-01] Critical
Web & API Penetration Test — Private Client Platform

Authorised black-box assessment of a production event-management and e-commerce platform. Identified 9 vulnerabilities (3 Critical, 4 High, 2 Medium) including privilege escalation, payment amount tampering, and PII exposure. Delivered a full report with risk ratings and remediation guidance, responsibly disclosed to the client.

Burp SuiteOWASP API Top 10Black-BoxREST API
Role escalation: a PATCH to /api/user/me accepted an unvalidated role: admin field — any authenticated user could self-promote to administrator with one request.

Payment manipulation: the payment flow trusted a client-supplied amount field. A purchase was completed for Ksh. 1 by intercepting and modifying the request before it reached the backend.

Mass assignment: the registration endpoint accepted an is_staff field directly from user input, allowing unauthenticated self-assignment of staff privileges.

Also chained: IDOR on ticket editing, stale tokens after logout, and BOLA/BFLA exposing marshal & vendor PII including national IDs.
[F-02] Critical
VulnWeb — Multi-Stack Web Application Assessment

Assessed five subdomains spanning ASP, PHP, ASP.NET, and HTML5 stacks. Chained missing authentication controls, SQL injection, and IDOR into full database compromise across multiple targets.

SQLMapBurp SuiteNucleiDirsearch
Auth bypass: admin' OR 1=1 -- against login forms on both ASP and ASP.NET targets granted full authentication bypass.

Full DB dump: automated SQLi via SQLMap against showforum.asp?id= and artists.php?artist= exposed complete user tables, including plaintext-adjacent credential hashes.

Missing auth: an empty-password submission on the HTML5 target's login form returned a fully authenticated session.

Also found: stored & reflected XSS, cleartext credential transmission, directory listing exposing a database schema file, and outdated PHP/ASP.NET versions.
[F-03] High
Flaws.Cloud — AWS Misconfiguration Chain

A six-level AWS security challenge escalating from a public S3 bucket to full IAM credential compromise via leaked git history, EC2 snapshot abuse, and instance metadata service exploitation.

AWS CLIIAMS3EC2 Metadata (SSRF)
L1–L2: public and "any authenticated AWS user" S3 bucket permissions exposed secret files with no authorisation checks.

L3: a bucket's exposed .git history contained a deleted-but-recoverable commit with a live AWS access key and secret.

L4: the leaked key enumerated a public EC2 snapshot; mounting the volume revealed setup credentials in a shell script.

L5: an HTTP proxy exposed the EC2 instance metadata service (169.254.169.254), leaking temporary IAM role credentials with access to the next bucket in the chain.
[F-04] High
BookStore — LFI to Root via Debugger PIN Bruteforce

Chained a source-code comment hint, API version regression, and local file inclusion to recover a Werkzeug debugger PIN — using it to spawn a reverse shell, then reverse-engineered a setuid binary's XOR check to escalate to root.

LFIGhidraFFUFReverse Engineering
Recon: an HTML comment on the login page hinted the Werkzeug debugger PIN lived in a user's bash history.

LFI: a deprecated v1 API endpoint (still live after a v2 migration) exposed an undocumented show parameter vulnerable to path traversal, leaking the bash history and the debug PIN.

Shell: the recovered PIN unlocked the Werkzeug interactive debug console, used to spawn a reverse shell as a low-privilege user.

Privesc: a setuid binary owned by root performed an XOR check on stdin input; reversing the XOR in Ghidra recovered the "magic number" and granted a root shell.
[F-05] Critical
CMSpit — Cockpit CMS CVE Chain to Root

Fingerprinted an outdated Cockpit CMS install via an exposed package.json, exploited a known username-enumeration CVE for initial access, then escalated through a file upload and a second CVE in ExifTool to full root compromise.

CVE-2020-35846CVE-2021-22204MetasploitPrivilege Escalation
Fingerprint: directory busting exposed package.json, revealing Cockpit CMS v0.11.1 — vulnerable to CVE-2020-35846 username enumeration.

Foothold: a malicious PHP file uploaded through the Cockpit interface returned a reverse shell as www-data.

Lateral movement: a .dbshell history file leaked a user's password, enabling SSH access and a user-flag.

Root: sudo rights on exiftool combined with CVE-2021-22204 (DjVu annotation command injection) produced a root shell via a crafted image file.
